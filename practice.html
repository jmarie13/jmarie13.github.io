Software Engineering and Design:

One of the most significant projects in my ePortfolio is the CornerGrocer, which I successfully migrated from C++ to Python. This project involved optimizing data structures by leveraging Python's collections.Counter, which enhanced both performance and simplicity. Additionally, I integrated MySQL database support, which improved scalability and data management. For example, I developed functions to fetch data from the database, allowing the program to display frequency lists and histograms with ease.

Beyond this project, I have consistently applied software engineering best practices across my work. In projects not included in my ePortfolio, I have employed principles like separation of concerns and modularity. For instance, in one project, I utilized the Data Access Object (DAO) pattern to decouple database access from the business logic, leading to more maintainable and testable code. These experiences reflect my commitment to writing clean, efficient, and scalable code.

Algorithms and Data Structures:

One of the cornerstones of my education has been the study of data structures and algorithms. The ability to select and implement the right data structure or algorithm for a given problem is a fundamental skill in computer science, and my coursework has provided me with a strong foundation in this area. Through projects like the CornerGrocer, I have applied these principles to optimize performance and ensure that my solutions are both efficient and scalable. This experience has reinforced the importance of algorithmic thinking in solving complex problems, a skill that is directly aligned with my career goals in data engineering.

My exploration of algorithms and data structures extended into the realm of reinforcement learning. In the Deep Q-Learning (DQN) Agent project, I worked on fine-tuning hyperparameters and implementing an early stopping mechanism that ensured the model's efficiency. By monitoring metrics such as loss, win count, and win rate, I was able to assess the effectiveness of the learning process and make adjustments to improve the agent's performance.

Outside of my ePortfolio, I tackled the Cartpole Problem using reinforcement learning techniques. This project involved modifying exploration factors, discount factors, and learning rates to optimize the algorithm's performance. Through iterative experimentation, I gained a deeper understanding of how these parameters influence the model's behavior, which enhanced my problem-solving skills. These skills are foundational to my work in software engineering, data engineering, and beyond.

Databases:

My experience with databases has been a key part of my work in building scalable and efficient systems. This is evident in the enhancements I made to the CornerGrocer project. By integrating MySQL database support, I enabled data persistence across multiple executions, which was crucial for handling larger datasets. I also created a SQL script to automate the creation of the database and tables, further streamlining the project's setup process.

Security:

Security has also been a recurring theme throughout my studies, emphasizing the need for vigilance in protecting sensitive data and ensuring the integrity of software applications. I have taken proactive steps to improve database security in my work. For example, I transitioned from using a JSON file for database credentials to employing environment variables. This change not only enhanced security but also demonstrated my ability to adapt and implement best practices. This focus on security aligns with my professional values and my commitment to developing software that protects users and their data.

This ePortfolio is a testament to the diverse range of skills and knowledge I have acquired during my time at SNHU. Together, they paint a comprehensive picture of my talents and readiness to contribute to the tech industry as a Data Engineer.
